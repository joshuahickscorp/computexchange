# SHIFT-REPORT · the interactive hero + accurate oracles (2026-07-02)

The public page hero is now a LIVE Three.js scene of a Mac Studio and an NVIDIA DGX Spark on a
desk, seen from a standing eye line looking down. Both devices are modeled from scratch in Blender
(no imported meshes, no photogrammetry, no trademarks). The Cycles stills and the real-time hero
come from ONE scripted scene, so the WebGL fallback is the same scene, not a placeholder.

## Phases and gate evidence

- **Phase 0 · teardown + scope audit** · `docs/SITE-REBUILD-T0.md`. The closed job-type contract
  was audited against the tree: six verified executors ship (embed, batch_infer, audio_transcribe,
  batch_classification, json_extraction, rerank), plus a metered general-compute BYO-container lane
  (not output-verified). Rendering/simulation are roadmap, not shipped · the copy reflects this.
- **Phase 1 · Blender env** · Blender 4.2.1 LTS, headless Metal GPU. Cube smoke render viewed.
- **Phase 2-4 · the scene** · `render/build_scene.py` (metric scale, tabletop 36 degree down
  camera, matte floor, key/rim/fill, AgX). Three verify loops logged in `render/NOTES.md` with a
  written PASS per device (Mac Studio ports + satin bead-blast; DGX Spark foam face + muted
  champagne) and per scene. Finals 1024 spp / 3200 px in `web/assets/site/`.
- **Phase 5 · glTF + maps** · `render/build_scene.py --export` emits a low-poly `oracles.glb`
  (flat foam plane, no 170k displacement) at **1.6 MB**. Foam normal/rough/AO maps generated by
  `render/foam_maps.py` (numpy Voronoi F2-F1, the same field the Cycles shader uses). Self-hosted
  Three r160 under `web/assets/site/vendor` (**1.4 MB**) · no CDN at runtime.
- **Phase 6 · integration** · `web/index.html` loads the hero via importmap + dynamic import.
  Live canvas with the Cycles still as the `<picture>` fallback. Copy rewritten to the compute
  scope; receipts dialog gains the general-compute row; 160 px section rhythm; phone hand-off
  unchanged.
- **Phase 7 · perf + honesty**:
  - **draw calls = 10** (gate: under 30), 7554 triangles · measured via `renderer.info` after a
    forced full frame. 60 fps is bounded by construction (10 meshes, pixelRatio capped at 2,
    render-on-demand so the page is idle when not interacting).
  - **hover raycast verified** · sweeping the cursor across the canvas lit the Studio label on the
    left third and the Spark label on the right (measured via dispatched mousemove).
  - **fallback verified** · forcing `WEBGL_lose_context` swapped the stage to `fallback` and the
    Cycles still to `display:flex`.
  - **dash gate clean** on every shift-authored file (grep U+2014/U+2013).
  - **asset handler** · `control/api.go` `handleSiteAsset` serves the scoped tree (png/js/glb/wasm/
    json) with a traversal guard, pinned by `TestSiteAssetType`.

## Honest limits (real-time vs Cycles)

The WebGL aluminum is matte PBR without the Cycles bead-blast microspecular, and the foam is a
baked normal map on a flat plane rather than true displaced geometry. Both are the standard
real-time simplifications; the Cycles still carries the full-fidelity version and is the fallback,
so no one is shown a lie. The gradient environment is a hand-authored dark equirect (PMREM), not a
photographic HDRI, per the void doctrine.
