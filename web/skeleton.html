Supplier — agent status

The supplier control surface is the local menu-bar app (macapp/), which reads ~/.compute-exchange/status.json written by the running agent. A browser can't read a provider's local file, so this tab documents that surface. Fields the agent already emits:
Agent stateactive / idle / paused / offline (status.json state)
Current jobstatus.json current_job / current_task_id
Resource limitsmemory headroom + max memory % (agent.toml)
Effective memoryavailable − reserved headroom (status.json effective_memory_gb)
Throttledstatus.json throttled + throttle_reason
Earnings / payoutstatus.json today_earnings_usd / balance_usd (placeholder here)
Logs / diagnostics~/.compute-exchange logs (placeholder — opened from the app)

This worker's live throttle/effective-memory state is also visible to operators on the Admin tab (/admin/workers).

Stuck jobs & incident response

      

        Stuck/long-running jobs surface in the Jobs view above (status running/verifying
        past their ETA). The control plane self-heals: stale-task requeue and
        straggler hedging run automatically. Manual runbooks (requeue, payout
        export, backup/restore, disaster recovery) live in docs/RUNBOOKS.md.
        Backups: make backup; restore is proven by make prove-local's disaster-recovery check.
