## The monument · OWED · viable, with one correction

Per-row `amount_usd` exists on `/admin/payouts` ✓, so summing unreleased liability is
possible from a single authenticated fetch.

- **Assumed:** payout statuses are `held / ready / released`; monument = held + ready.
- **Found:** the real status set is `pending / held / released / clawed_back`
  (control/payment.go). There is NO `ready`. Money owed but not yet paid = `pending`
  and `held`. `released` = paid out; `clawed_back` = reversed fraud.
- **Decision:** the OWED monument sums `amount_usd` where `payout_status ∈ {held, pending}`,
  excluding `released` and `clawed_back`. Subline reads `held + pending · released and
  clawed-back excluded`. Rendered only from a successful authenticated fetch; absent an
  admin key or on any error the slot renders nothing (not a dash, not zero).
